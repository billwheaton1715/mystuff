This is the the mystuff README.md file.

To make a repository and have it synch with what is on the server, you have to actually create a repository right on the server. With GitHub, you can log into the web interface and make one there. Once you do, it gives you the right strings you need to use in the local Git bash shell to create a local repository, create a readme file there, add it to staging, and commmit it locally. Then you do a `git remote add origin ` to give the alias 'origin' to the url. Then you `git push -u origin master` to push the readme to the server. At that point you can get onto github again and see the README.md file. (this file)
